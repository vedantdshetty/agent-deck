Produces:

- A WatcherAdapter-interface-compliant SlackAdapter that routes Slack events
  via the ntfy bridge rather than native Slack Socket Mode (ADAPT-04 rule).
- Event normalization with `Sender` formatted as `slack:{CHANNEL_ID}` so
  the router's `clients.json` can key entries on channel IDs (mirroring the
  format used by the Wave 2 `watcher import` command).
- Support for both v1 legacy plain-text payloads and v2 structured JSON
  payloads emitted by the Cloudflare Worker ntfy bridge.
- Thread reply detection: when the v2 payload's `thread_ts` differs from
  `ts`, the emitted Event carries a `ParentDedupKey` of the form
  `slack-{CHANNEL_ID}-{THREAD_TS}` so the engine can look up the parent's
  session_id for thread routing (ADAPT-04 rule).
- Reconnect with exponential backoff (2s initial, 2x factor, 30s cap)
  reusing the NtfyAdapter pattern shipped in Plan 14-01.

No code changes in this PLAN: all implementation shipped on 2026-04-10.
